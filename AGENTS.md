# AGENTS.md

This file guides AI coding agents (Claude Code, Codex, Cursor, Copilot, and others) working
in this repository. It is loaded into agent context automatically — keep it concise.

## Overview

`@onflow/passkey-wallet` is a demo WebAuthn passkey wallet for Flow, implementing the
[FCL Wallet Provider Spec](https://developers.flow.com/build/tools/wallet-provider-spec)
and FLIP-264 (WebAuthn Credential Support). It is a single-page Vite + React 19 +
TypeScript app exposing a host demo (`/`), a wallet connect popup (`/authn`), and a
wallet sign popup (`/authz`). Demo only; not production software (see README).

## Build and Test Commands

- `npm install` — install dependencies
- `npm run dev` — start Vite dev server on `http://localhost:8710`
- `npm start` — alias for `npm run dev` (package.json `"start": "npm run dev"`)
- `npm run build` — run `tsc -b` then `vite build`
- `npm run preview` — serve the production build (Vite default port)
- `npm run serve` — serve the production build on port 8710
- `npm run lint` — run ESLint (flat config, ignores `dist`)

There is no configured test runner: `src/wallet/__tests__/authz-signature.test.ts` uses
jest-style globals and CommonJS `require`, but neither `jest` nor `vitest` is a dependency
and `package.json` declares no `test` script. Do not assume `npm test` works.

## Architecture

Client-only SPA; no backend in this repo. Routing is hand-rolled in `src/routes.ts`
(popstate + `window.history.pushState`), not React Router.

- `src/main.tsx` — React root, renders `<App />` in `StrictMode`
- `src/App.tsx` — top-level view switch over `/`, `/authn`, `/authz`; owns the WebAuthn
  signing flow, FLIP-264 extension data assembly, and FCL View messaging
- `src/routes.ts` — `ROUTES`, `pathToView`, `viewToPath`, `navigateTo` helpers
- `src/pages/HostHome.tsx` — host demo; wraps children in `FlowProvider` pointed at
  `https://rest-testnet.onflow.org` (testnet) and uses Flow React SDK hooks
  (`useFlowCurrentUser`, `useFlowMutate`, `useFlowTransactionStatus`)
- `src/pages/wallet/AuthnView.tsx`, `src/pages/wallet/AuthzView.tsx` — popup views
  implementing the FCL View contract (READY / READY:RESPONSE / RESPONSE)
- `src/wallet/webauthn.ts` — WebAuthn create/get, base64url helpers, `derToP256Raw`
  (DER → 64-byte raw signature)
- `src/wallet/encode.ts` — `encodeMessageFromSignable`, transaction payload/envelope
  RLP encoding with FLIP-264 `sigExt` bytes
- `src/wallet/messaging.ts` — cross-window postMessage helpers for View protocol
- `src/wallet/provision.ts` — account provisioning via
  `https://hardware-wallet-api-testnet.staging.onflow.org/accounts`
- `src/wallet/types.ts`, `src/ui/types.ts`, `src/types/shims.d.ts` — type declarations
- `src/components/Nav.tsx` — navigation component

## Conventions and Gotchas

- **Dev/serve port 8710 is set via npm scripts, not `vite.config.ts`.** `vite.config.ts`
  only sets `base: ""` and registers the React plugin. Running `vite` directly without
  `--port 8710` will bind to Vite's default port.
- **Network is hardcoded to testnet** in `src/pages/HostHome.tsx` (`flowNetwork: "testnet"`,
  `accessNodeUrl: "https://rest-testnet.onflow.org"`) and in `src/wallet/provision.ts`
  (hardware-wallet-api staging). Change both when retargeting.
- **RP ID is `window.location.hostname`** (README). Passkeys created on one host cannot
  be used from another; running under a different host invalidates stored credentials.
- **Signatures follow FLIP-264:** the Flow `signature` field is raw P-256 `r||s`
  (64 bytes); WebAuthn materials ride in
  `extension_data = 0x01 || RLP([authenticatorData, clientDataJSON])`. See `src/App.tsx`
  around lines 386–405 and `src/wallet/encode.ts` `sigExtBytes`.
- **Key algorithm is P-256 / SHA2-256** (README) — the Flow account key must be created
  with the matching sig/hash algorithms.
- **TypeScript is `strict`.** The root `tsconfig.json` is a project-references file
  (`"files": []`, references `tsconfig.app.json` and `tsconfig.node.json`). The effective
  `strict: true` for `src/` is in `tsconfig.app.json` (line 20), and the same flag applies
  to `vite.config.ts` via `tsconfig.node.json` (line 18). Both also enable `noUnusedLocals`,
  `noUnusedParameters`, `erasableSyntaxOnly`, `noFallthroughCasesInSwitch`, and
  `noUncheckedSideEffectImports`. `tsc -b` runs as part of `npm run build`, so type errors
  break the build.
- **ESLint uses flat config** (`eslint.config.js`) with `typescript-eslint`,
  `eslint-plugin-react-hooks`, and `eslint-plugin-react-refresh`. The `dist/` directory
  is globally ignored.
- **Demo only** (README): no custody, recovery, or production UX. Do not add features
  that imply otherwise without discussing.
